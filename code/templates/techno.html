Стек технологий

Поговорим о тех технологиях, которые использовали при создании

Бэкенд:


- Python 3.10 — основной язык программирования, на котором написана вся серверная логика приложения.
- FastAPI — современный высокопроизводительный веб-фреймворк для построения REST API. Используется как основа приложения, обеспечивает автоматическую генерацию OpenAPI-документации и валидацию данных.
- Uvicorn — ASGI-сервер для запуска FastAPI-приложения. Обеспечивает асинхронную обработку HTTP-запросов.

.

База Данных:


- PostgreSQL — реляционная СУБД для хранения данных пользователей, рецептов, отзывов и метрик.
- SQLAlchemy — ORM-библиотека для работы с базой данных на уровне Python-объектов. Позволяет описывать модели данных как классы и выполнять запросы без написания «сырого» SQL.
- Psycopg2 — драйвер-адаптер для подключения Python-приложения к PostgreSQL.

.

Аутентификация и Безопасность:


- AuthX — библиотека для реализации JWT-аутентификации через cookies. Управляет генерацией и валидацией токенов доступа.
- Bcrypt — библиотека для хеширования паролей. Обеспечивает безопасное хранение паролей пользователей в базе данных.

.

Шаблоны и Фронтенд:


- Jinja2 — шаблонизатор для рендеринга HTML-страниц на сервере. Используется для генерации динамического контента.
- HTML/CSS/JavaScript — клиентская часть интерфейса, обслуживаемая через StaticFiles FastAPI.

.

Перевод и Локализация:


- Deep Translator — библиотека для автоматического перевода текста. Используется для перевода рецептов и ингредиентов с английского на русский язык (интеграция с Spoonacular API).

.

Тестирование:


- Pytest — фреймворк для написания и запуска юнит- и интеграционных тестов.
- HTTPX — асинхронный HTTP-клиент для тестирования API-эндпоинтов.

.

Контейнеризация и CI/CD:


- Docker — платформа для контейнеризации приложения. Позволяет упаковать приложение со всеми зависимостями в изолированный контейнер.
- GitHub Actions — сервис для настройки CI/CD-пайплайна (автоматическая сборка, тестирование и деплой при пуше в репозиторий).

.
